## 1. 코드 분석 핵심 발견

### 1-1. 데이터 소스 = 네이버 HTML 스크래핑 (단일 장애점)
- `paser/ExchangeInfo.java`: `https://finance.naver.com/marketindex/exchangeList.nhn` + `https://finance.naver.com/marketindex/` 두 페이지를 JSoup으로 스크래핑. 공식 API 없음.
- `ExchangeParser.java`: `"tbody tr"` 셀렉터 → 공백 split. "남아프리카 **공화국**" 문자열 감지 → ZAR 강제 등 하드코딩 방어. JPY는 6개 필드 `/100` 정규화. **IOException은 삼켜져** 실패가 위로 전파 안 됨 → 리뷰의 "정보 안 뜸 + 광고만 뜸"과 직결.
- 그래프는 네이버 정적 이미지(`ssl.pstatic.net/imgfinance/...`), 국기는 `flagcdn.com`.

### 1-2. 알람 시스템 — API 레벨별 3중 분기, 절반이 죽은 코드
- `RestartService.kt:44-79` `serviceStart<T>()`:
  - **API 31+ (대다수 사용자)**: 제네릭 T(AlarmService)를 무시하고 `BackupWorker`를 WorkManager PeriodicWork(최소 15분)로 등록.
  - API 26~30: `AlarmService` FGS + CountDownTimer 루프.
  - API 25-: 일반 startService.
- 즉 **API 31+에서 `AlarmService.java`(287줄) 전체 + `RestartAlarm.kt`의 1초 setRepeating 워치독은 죽은 코드**. 워치독은 OS 스로틀(~60s)로 WorkManager REPLACE 재등록만 반복 → 순수 배터리 낭비.
- `AlarmService.java:172-191`과 `BackupWorker.kt:95-103`에 **판정 로직 완전 중복** (알림 ID 2130까지 동일).
- 포그라운드 억제 판단 `SystemUtil.isAppForground()`가 deprecated `getRunningTasks()` 의존 → 최신 OS에서 신뢰 불가.
- `SCHEDULE_EXACT_ALARM` 권한 선언만 있고 실사용 없음 (죽은 권한).
- 배터리 최적화 예외 유도/제조사 대응 없음.

### 1-3. Realm — 업데이트 한 방에 사용자 데이터 유실 구조
- `MyApplication.kt`: `deleteRealmIfMigrationNeeded()` → **스키마 변경 시 사용자 알람/설정 전량 삭제**. 대폭 개선 시 최우선 함정.
- `allowWritesOnUiThread(true)` + RealmController의 **포지션(인덱스) 기반 update/delete** → 순서 변동 시 엉뚱한 알람 수정/삭제 위험.
- 읽기 판정에 `executeTransaction`(쓰기 락) 오용.

### 1-4. 의존성 — "표면만 최신" 레거시
- 툴체인: AGP 8.3.2, Kotlin 1.9.25, compile/targetSdk 35 (최신) ✅
- 실질: JSoup 1.14.3(2021), Glide 4.12.0(2021), **구 support `constraint-layout:2.0.4`**, `firebase-ads:23.5.0`(구형 아티팩트), realm-android-adapters jitpack 스냅샷, 구형 databinding compiler kapt, 루트에 jcenter() 잔존, Retrofit 선언만 되고 미사용 ❌
- `minifyEnabled false` (릴리스 축소/난독화 없음), 테스트 보일러플레이트뿐, **values-en 없음(한국어 전용 — repo CLAUDE.md의 "영어 지원" 기술은 오류)**, 홈 위젯 없음.
- Java 25 / Kotlin 18 파일 — 핵심 도메인은 전량 Java, 최근 패치만 Kotlin.

### 1-5. 수익화/정책 리스크
- 배너 + 스플래시 전면 (AdChecker MAX_COUNT=4 게이팅). 네이티브 광고는 상수만 존재.
- **UMP(GDPR) 동의 흐름 전무** — MobileAds.initialize 무조건 호출. 정책 리스크.
- AdMob App ID: `ca-app-pub-1908860913688060~...` — **khy(pub-7340…6505)·khw(pub-8322…2133) 어느 쪽도 아닌 제3(구) 계정**. 개선 시 계정 정리 필요.
- Remote Config는 `ad_once` 1개만 사용. 카카오 공유 존재.

## 2. 시장 리서치 핵심 (국내)

### 2-1. 시장 컨텍스트
- 고환율 뉴노멀(1,400~1,500원대, 2026.6 장중 1,561.5원) → "언제 환전하나" = 알림 수요 구조적 상승.
- 환전 채널은 트래블카드로 재편(선호 53.3% vs 현금환전 11%). 하나 트래블로그 가입 1,000만.
- 환테크 대중화: 알림 + 자동환전 병행이 재테크 표준 조언.

### 2-2. 경쟁 구도
- **핀테크/은행**: 토스 외화통장(목표환율 자동환전, 수익률 알림, 실시간 위젯), 트래블카드 3대장(수수료 0 기본값化). 전부 **자사 가입 전제** → 비가입 중립 도구는 공백.
- **포털**: 네이버 환율(하나·신한 고시 기반, 차트, 목표가 푸시)은 "조회"는 강하나 위젯·정밀알림·은행비교 없음.
- **Play 전문 앱**: SmartWho ALL 환율(500만+, 4.82), 오늘의 환율(50만+, 4.06), SangGeon 환율계산기(50만+, 4.25, 4종가+우대율), makeappeasy 환율 알리미(5만+, 4.71, 위젯 특화), XE(1,000만+).

2. **소형/커스텀 위젯** (1x1·2x1, 다크, 등락 색, 투명도, 교차환율) — 명시적 공급 공백

## 3. 데이터 소스 리서치 결론 (2026-07-22 curl 실검증)

### 3-1. 주 소스: 네이버 비공식 JSON API ✅ 검증됨
- `GET https://api.stock.naver.com/marketindex/exchange/FX_USDKRW` → 하나은행 고시회차(당일 624회차 확인), closePrice, 등락, marketStatus. **`FX_USDKRW_SHB`로 신한은행 고시도 동일 구조 제공**.
- `GET https://api.stock.naver.com/marketindex/exchange/FX_USDKRW/prices?page=1&pageSize=10` → **매매기준율 + 현찰 살/팔 + 송금 보낼/받을 4종이 전부 JSON** + 일별 히스토리 페이지네이션(차트 데이터 해결).
- 무키·무쿠키 GET. 현 JSoup과 동일 원천(하나은행 고시)을 구조화 JSON으로 획득.
- 리스크: 비공식 → **자체 서버(apis-py) 경유 프록시+캐시 필수** (스키마 변경을 서버에서 흡수).

### 3-2. 폴백 1: 한국수출입은행 공식 API
- ⚠️ **2025-06-25부로 도메인 변경: `oapi.koreaexim.go.kr`** (구 www 도메인 점진 종료).
- 무료 일 1,000회, 하루 1회 고시(영업일 11시 전후, 이전엔 null), 주말 없음. `deal_bas_r`/`ttb`/`tts` 제공, **현찰가 없음**. 키 2년 경과 자동 파기(RESULT=3) 모니터링 필요.

### 3-3. 폴백 2: frankfurter.dev(무키·무쿼터, ECB 일1회) 또는 fawazahmed0 CDN(무제한 정적 JSON). exchangerate.host는 2023-09 유료화됨. 두나무 비공식은 금일 접속 실패 — 후순위.

### 3-4. 권장 아키텍처
```
앱 → 자체 서버 API(apis-py 신규 exchange 도메인: 네이버 JSON 수집·정규화·캐시,
     실패 시 수출입은행 → frankfurter 폴백, source+갱신시각 필드 포함)
   + 서버사이드 alert 엔진(고시회차 폴링) → FCM 푸시
앱은 마지막 응답 로컬 캐시 → 오프라인 표시/계산
```
- Rate alert의 업계 표준은 서버 모니터링+푸시(XE/Wise/Revolut 전부). 로컬 WorkManager 폴링(15분 제약)은 오프라인 보조로 강등.

### 3-5. 글로벌 앱 수익모델 표준
무료+광고 → 구독/단건으로 ①광고 제거 ②백그라운드 자동 갱신 ③고급 차트 ④알림 고급 기능. (송금 수수료 모델은 비현실적.)
